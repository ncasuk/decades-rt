Description

The Honeywell H-423 Inertial Navigation Unit (INU) is mounted on the base of the rear core console. Its main purpose is the measurement of aircraft velocity and attitude for use in calculating winds; aircraft attitudes are also required for some radiation measurements. Other necessary parts of the system are the baro signal which is supplied from the turbulence probe, located in the nose bay, the INU Interface Unit (IIU) on the rear core console, which interfaces the INU to the DRS, and HORACE which is required when starting the INU.

The INU uses high precision accelerometers to sense accelerations, which are then integrated to provide velocities. Ring Laser Gyros are used to sense angular rotations, which are then integrated to provide attitudes. The system is a strapdown system with the sensors rigidly fixed to the aircraft. Before the system can make any measurements it requires a gyro-compassing period of about 9 minutes during which it senses the earth's rotation in order to determine the direction of true north. Attitudes are measured to an accuracy of about 0.1 degree, and velocities to 2 nmi/hr over a long flight; corresponding position errors can be up to 25 nmi.

The INU is powered by 115V 400Hz, but switches automatically to 28V DC if the 400Hz is lost. The 28V is supplied by a power supply in the IIU cardframe running off the UPS maintained 230V. The INU requires fan cooling which also runs off 230V 50Hz. An on/off switch on the INU control panel is used to start and stop the system; note that the flight manager's INU control panel is replicated on the aft core console, behind the flight manager's seat.

All control and communication for the INU takes place using two MIL-STD-1553B data networks. These networks work independently; although the INU cannot handle control commands on both networks simultaneously this is not a practical problem. Only one network is now used. Note that the 1553B networks uses blue coax cables which are easy to identify. Each network has a second redundant bus, which can be used if the first fails (by manual reconnection); also note that the IIU will work with either network if necessary. The second network is used solely by the INU Interface Unit, which provides the interface between the INU and the DRS.

The IIU's main function is to request data from the INU 32 times a second, to repackage the data, and to send it to the DRS's IIUDLU as seven 32 Hz parameters, 163 to 169. The IIU also reads the baro signal and passes it to the INU at 16 Hz. The IIU has a Serial Interface (ISI) which will accept RS232 messages for onward transmission to the INU, and return any messages received from the INU back through the RS232 port; although not intended to work at a high data rate, the ISI provides the means to interrogate the INU for any item of information, and also provides a means of starting up the INU using HORACE. The ISI is used by HORACE, via the HORNET terminal server, to allow any INU message to be displayed for diagnostic purposes, and to provide emergency alignment of the INU. The IIU cardframe also contains the IIUDLU which packages the INU data into Ethernet packets and sends them to HORACE.

The Melchor PSU is driven by the UPS and provides backup 28V for the INU. The Trivolt PSU is driven by the PSU and powers the IIU and the IIUDLU. Note that the IIUDLU needs to be connected to the DLU umbilical panel, but the associated CB on this panel does not need to be made since the umbilical is only providing Ethernet and timing signals, not power.

Although the INU contains a sophisticated navigation computer, we are not currently making use of this, and there is no direct control or display of INU data provided for the aircrew.

Pre-flight checks
  1. Indicators and parameters as for Normal Operation.
  2. INU latitude and longitude accurate to 1 nmi ± 0.8 nmi/hr since alignment.

In-flight checks
  1. Indicators and parameters as for Normal Operation.

Post-flight
  1. Ensure INU remains on and recorded for at least 10s after aircraft stops, and note the aircraft's actual stopped position.
  2. INU OFF, using INU remote panel on/off switch and ensure the UPS is not switched off for 10s.

INU Panel

The INU Control Panel, on the Flight Manager's console and also replicated on the rear core console behind the Flight Manager's seat contains, the following indicators and controls:

  1. INU ON/OFF switch - Switches the INU between OFF and STANDBY modes; allow 10s between switching OFF and switching off the UPS. This switch also starts/stops the cooling fan.
  2. DC ON lamp (green) - ON = 28V DC is available to the INU, OFF = No maintained 28V
  3. Attitude lamp (green) - ON = INU is in GC_ALIGN or NAVIGATE mode, OFF = INU is OFF, in STANDBY, or no 115V
  4. 115V ON lamp (green) - ON = 115V 400Hz is available to the INU, OFF = No 115V

Switching on
  1. CB's set as in Circuit Breakers Required section.
  2. INU ON, using INU control panel. Toggle OFF then ON if already in ON position. Note that ON on either panels constitutes ON.
  3. Press IIU reset button.
  4. Note that the INU must be aligned after being switched on.

Alignment

Note:

  1. Alignment (also known as gyro-compassing) takes about 9 minutes.
  2. Alignment can only be done when the aircraft is stationary.
  3. Attempts to align while the aircraft is in flight will be rejected.
  4. If the aircraft moves before alignment is complete INU measurements will be of reduced quality.
  5. Once alignment is complete, NAVIGATE will be entered automatically if the aircraft moves before you change mode to NAVIGATE.
  6. The latitude and longitude entered during alignment should be accurate to 1 nmi.

The following method of alignment uses HORACE and requires HORACE, HORNET, and the IIU to be running:

  1. Ensure HORACE, HORNET, and the IIU are running, and the DRS is recording data.
  2. Switch the INU ON. If it is already ON then switch OFF with INU on/off switch, wait 10 seconds, and switch ON again.
  3. Select the INSTRUMENTS/INU/INU ERRORS AND STATUS option on HORACE and check that the INU mode is STANDBY.
  4. Select the INU/SERIAL INTERFACE/EMERGENCY CONTROL option, and then the CHANGE MODE TO GC_ALIGN option.
  5. Enter current position as GPS, or any other accurate position; when these are entered the INU is put into GC_ALIGN mode by HORACE.
  6. Select the INU/INU DATA option and check the INU mode is now GC_ALIGN and the latitude and longitude are correct; if not go to step 4 and try again.
  7. Select the INU/INU ERRORS AND STATUS option and observe the DEGRADED NAV RDY bit of the I01-01 word (other bits in this word may also be changing). This bit sets about 1.5 minutes after entering GC_ALIGN. Wait until this bit clears (about 8.5 minutes after entering GC_ALIGN).
  8. Select the INU/SERIAL INTERFACE/EMERGENCY CONTROL option, and then the CHANGE MODE TO NAVIGATE option.
  9. Select the INU/INU ERRORS AND STATUS option and check the INU mode is NAVIGATE.

Possible problems

Switching off - Given that the INU can not be aligned in the air there is little point in switching it off in the air unless there is a major problem - eg overheating due to failed fan or high ambient temperature. Even if it is not functioning fully it may be possible to retrieve some data post-flight.

Switching on - If the INU has been switched off for any reason, eg loss of both 28V and 400Hz or accidental use of the control panel on/off switch, there is no point in starting it up in the air. If power returns it will not try to restart provided you don't toggle the on/off switch OFF then ON.

Overheating - The INU should be able to run for up to five minutes without the fan. Beyond this everything depends on the ambient temperature, but it is suggested that the INU should be shutdown after five minutes without the fan unless there is a strong scientific requirement, or there is an immediate prospect of the fan starting again. The INU claims it will shutdown if it overheats without damage to the unit, but this has not been tested, yet.

Alignment - When HORACE is used to start the INU alignment, it sometimes fails to put the INU into GC_ALIGN and sometimes fails to use the correct latitude and longitude. In both cases retrying the operation normally works.

INU interface unit (IIU) - If any problems are suspected with the IIU it can be reset or switched OFF and ON at any time, and should recover without further attention. The INU will show invalid vertical data for 90 s after the IIU is switched on.

Baro signal - The loss of baro information to the INU will affect its vertical channel, ie height and VZ, but not its horizontal measurements nor its roll, pitch and heading measurements. Without a baro input the INU will simply use inertial information for the first five minutes, after which it assumes the baro height is fixed at the current height.

HORACE - Since INU data are logged by the DRS, there are no special processes on HORACE which need to be checked for correct operation. The H_INU process on HORACE only supports the INU Serial Interface (ISI). It lies dormant except when the INU first goes into NAVIGATE or if someone uses the HORACE ISI display option to align the INU or display special data. It can safely be killed if necessary - H_STOP H_INU - without any effect on the recording of INU data.

Other items (for interest):

VX,VY,VZ

Aircraft velocities in the platform frame - VX doesn't always point north, and VY typically is positive west not east. VZ is +ve up.

VN,VE

Aircraft velocities computed from VX and VY.

Lat and Lng

Only as good as initial position plus 2 nmi/hr possible error.

Alt and Height

Mainly based on pressure altitude.

G/SPD and Track

Computed from VX and VY; track is true not magnetic.

Roll, pitch, true heading, magnetic heading, platform azimuth

All with respect to the ground not the aircraft. Note that the 'platform' is aligned with north during gyro-compassing but drifts as the aircraft moves east or west.

Wander angle

The drift of the 'platform', ie the difference between the platform azimuth and true heading.

Roll rate, pitch rate, yaw rate

All with respect to the aircraft not the ground.

CNEXX, CNEXY, CNEXZ

Direction cosines used to transform between the navigation frame and the earth frame. Therefore always in range ±1.

AX, AY, AZ

Aircraft accelerations in the platform (X,Y,Z) frame.

A/Forw, A/Port, A/Vert

Aircraft accelerations in the aircraft frame - what you feel.

IIU version

The software version of the IIU software, likely to stay at 0.
