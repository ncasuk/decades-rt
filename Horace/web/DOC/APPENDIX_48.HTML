Satcom-C System

Circuit Breakers Required

  1. 28V DC, Satcom-C - Power for the Satcom-C system

Normal Operation
  1. Satcom transceiver:
    • Power lamp (green) - ON
    • Login lamp (green) - ON (may be flashing)
    • Send lamp (green) - OFF (unless sending a message)
  2. HORNET:
    • Port 7 lamp (green) ON and flashing every 8 s.
  3. HORACE Satcom menu, Option 7, Line 1:
    • Mode - Free
    • Ocean - E or W Atlantic, Indian, Pacific
    • Sync - Yes
    • Sig - 5
    • GPS - 2-D or 3-D

Description

The Satcom-C system comprises a jet blade antenna mounted on the top of the aircraft, aft of the wings, an amplifier unit mounted in the roof of the aircraft immediately below the antenna, and a transceiver unit mounted in the aft core console alongside the INU. The whole system is powered by a single 28V DC supply, which is not maintained across power changeovers. A red flashing light and reset switch is provided on the forward core console to indicate the reception of new messages.

The system provides low rate (600 baud) message transfer through the four INMARSAT-C geostationary satellites. Data are transferred between the aircraft and sources or destinations on the ground as messages through land stations (normally Goonhilly in our case) on a store and forward basis so that four or five minutes can elapse between a message being transmitted and it being received.

All control and communication with the Satcom-C transceiver is handled by HORACE using the H_SATCOM process which communicates with the transceiver via an RS232 link from HORNET. H_SATCOM also collects messages for transmission from the display and AVAPS processes, and provides status information for display on PCs connected to HORACE. There is normally no need or capability for users to talk to the transceiver directly, although this can be done with a VDU or the Toshiba portable PC in an emergency. Note that if the Satcom-C system is powered up it will receive messages autonomously even if HORACE is not running, and these will be retrieved by H_SATCOM when HORACE starts.

All messages received by the Satcom-C system, except special system messages beginning with a '|', are stored in files on HORACE and can be displayed on any HORACE display using the list and show received message options of the H_DISPLAY Satcom menu. When a new message is received H_SATCOM also switches on the flashing light on the Flight Manager's console; this light will remain flashing until the Flight Manager choses to clear it by using the reset button. The flashing light is driven by RS232 from HORNET port 6.

HORACE has a list of standard message addressees, and messages can also be sent to other fax or mail addresses if necessary. HORACE puts default 'From:' and 'To:' headers on the message which the Flight Manager can elect to use, change or dispense with. The Goonhilly and other groundstations pass all messages to MRF, from where they are forwarded to the required destination. Messages for sending are queued on HORACE, with messages from the Flight Manager taking precedence over drop-sonde messages, and earlier messages being transmitted before later ones. Option 10 of the Satcom menu gives the Flight Manager the option to cancel queued messages, or to retransmit stopped or failed messages. Any messages pending when HORACE or the Satcom-C system is switched off will not be transmitted on switch on, unless the Flight Manager explicitly resubmits them.

The system transmits messages one at a time, except for drop-sonde messages, which may be packed together if more than one is available for transmission. The maximum message length the Flight Manager can send is 20 lines, but received and transmitted messages of any length can be displayed. The maximum message length that the system can receive is 50 lines, and the maximum transmitted message length is 15000 characters (eg multiple sonde messages).

The transceiver contains an in-built GPS system for computing the Doppler shift to apply to transmitted signals; this also provides, every eight seconds, a measurement of position, height, speed and track. This information, together with the flight number, is normally transmitted to MRF every 15 minutes on the quarter, when airborne. Additional position reports may be sent at takeoff and landing.

The Satcom-C transceiver has to be logged into one of the four INMARSAT-C satellites in order to send or receive messages. These are positioned over the West and East Atlantic, the Indian, and the Pacific oceans such that the system should always work to ±70 deg latitude, and up to ±80 deg for most longitudes. Do not expect the system to work while in a hangar! In many parts of the world more than one satellite will be visible. When the aircraft is flying, H_SATCOM always tries to keep the transceiver logged into the nearest satellite, ie the one with the highest elevation, and does not try to change it even if it goes out of sight due to manoeuvres or the track of the aircraft putting the tail in line of sight to the satellite; this is because it takes at least a minute to login into another satellite by which time the best satellite may be visible again. When on the ground the system tries all possible satellites, since the optimum one may be obstructed by hangars etc. Even if an INMARSAT-C satellite is visible, there may not be enough GPS satellites visible to determine the aircraft position and speed; in this case the system may still receive messages, but will be unable to transmit them.

Pre-flight checks
  1. Satcom-C transceiver and HORNET indicators and displays as for Normal Operation.
  2. HORACE Satcom menu, Option 7 - System should be responding, mode should be "Free" or possibly "Login". Provided the antenna is in line of sight of a satellite one of the four ocean regions should be displayed and GPS should be 2-D or 3-D.

Post-flight
  1. Aft core console EDB, 28V DC, Satcom-C - OFF

Receiving/reading messages

Option 1 of the Satcom menu provides a list of all messages received by the system ending with the latest message. This list is updated when a new message is received. Each line contains a message number, date, time, message size, and part of the first line of the message, as follows:

00012  06-JAN-95  12:19:09    61  This is the first line of a message to XV208 w

Option 2 allows each received message to be displayed. The header line is displayed, followed by the contents of the message. The latest message is always displayed first, and keys D and E allow stepping through other messages. For messages of more than 20 lines key H allows the message to stepped through.

Option 3 provides a list of all messages passed to H_SATCOM for transmission. The entries in this list are updated whenever there is a change. Each line contains a message number, date and time the message was prepared, message size, message destination, message status, and message source, as follows:

00001  09-JAN-95  15:08:07    58  MRF fax at Farnborough         Successful   FL

Message status has the following meanings:

Queued

Message has been put on the H_SATCOM queue for transmission

Submitted

Message accepted by transceiver for transmission

Issued

Message transmission started by transceiver

Successful

Message successfully reached the groundstation, eg Goonhilly

Transfer failed

Message failed to transfer from H_SATCOM to transceiver

Submit failed

Message not sent because no LES or another active transmission

Issue failed

Message for transmission rejected by transceiver

Send failed

Message failed to reach ground station

Unknown

Message was issued but no other information is available

Stopped

Message has been stopped by Flight Manager or H_SATCOM

Compacted

Message has been included in a larger message (sondes)

Message source has the following meanings:

FL

From Flight Manager

NC

A set of compacted NAVAID messages

NV

From NAVAID system

TA

Turnaround message sent from the ground

FT

File transfer request sent from the ground

PR

Position report

Sending messages

Option 9 gives the Flight Manager the ability to send messages. This is done in a number of stages as follows:

  1. A destination is selected from the list displayed. If required, fax, phone numbers, etc will be need to be entered at this stage.
  2. A "From:" entry for the message is entered, or the default used, - use a space if no entry is required.
  3. A "To:" entry for the message is entered, or the default used, - use a space if no entry is required.
  4. The message is typed in, terminated with a CTRL/Z.
  5. The whole message is displayed and confirmation sought to send it.
  6. The message is put on H_SATCOM's message queue.

The typical destination list is shown below. The first 2 entries may change to meet requirements, but they are characterised by not requiring any numbers or addresses to be entered by the Flight Manager; the necessary routing of the message is worked out when it gets to the MRF base station. Option 6 if provided for testing of the system. Entries 3 to 5 will require the Flight Manager to enter specific fax or phone numbers (including dialling code from MRF) or other addressing information. The emergency routing option is only for use when the MRF base station in not available, and once again will required explicit addressing information from the Flight Manager.

Message destinations:

  1. Return to Satcom menu
  2. Normal routing via Met Office
  3. Echo back to G-LUXE Satcom-C
  4. Specified CONDOR file name
  5. Specified Internet address
  6. Emergency routing, bypassing Met Office

Option 10 gives the Flight Manager the ability to stop a message on the queue or to abort a message which is actually being transmitted. The message remains on HORACE and can be resubmitted later. The Flight Manager can also resubmit any message listed in the transmission log provided it is not currently being transmitted; in this case the message number, date and time in the log retain their original values and a new entry is not made.

Dropsonde messages

A separate process, H_NAVAID, collects dropsonde messages from the NAVAID PC and H_SATCOM sends them from the Satcom-C system to the Met Office's Met Data Bank, via MRF. To reduce messaging time, multiple sonde messages may be compacted into a single large message before transmission. No intervention should be needed from the Flight Manager, but the status of any sonde messages should be monitored using option 3, and the contents of the messages with option 4, of the Satcom menu. If a message tranmission fails, the Flight Manager should resubmit it using option 10.

Interpretation of HORACE Satcom displays

Options 5 to 8 of the HORACE Satcom menu provide detailed information on the status of the Satcom-C transceiver and are explained below; these options need not be used routinely, although the first line of the option 7 display provides a concise summary of the status of the system. With all these options there may be a little delay while the information is collected, whereafter is is updated about every eight seconds, or when there is a significant change.

Hardware:

Clock

Resolution of 1 min. Set to GPS time every 4 hours. Should be right.

Synth's

Synthesizers in lock. Should all read yes.

LO 1/2

Local osc output level. Should be 500 to 5000 mv. LO2 unused.

Cor A/C/R

Frequency tracker level actual/centre/range. Should have C-R<A<C+R.

Dif/Temp

Freq diff from channel freq. Should be between ±150Hz. Temp near oscillator. Should be 10 to 15 deg above ambient.

Antenna receive current. 105mA/0/Too high = good/no antenna/short.

TX

Last transmit current. Should be in range 2000 to 3000 mA. 0 until a message has been transmitted.

AGC

3000mv for good antenna signal, 2000mv if antenna disconnected.

FIFO

Transmission queue. Should always be yes.

R

Viterbi Renormalizations. A large number means weak signals.

C-B

Viterbi cycle slips and bit slips. Should both be 0 if good signal.

B

Bulletin board check should be 1. Bull count increments every 8.64s.

S

Signal strength from 0 to 5. 3 or more needed for good comms.

Synchronization

Yes if transceiver can receive and decode satellite signal.

Logged in

Yes if have not logged out from a ground station.

TDM type

Channel in use. NCS (Network Coordinating Station) or LES.

TDM channel number

8000 to 14000, corresponds to transmit/receive frequency.

Current channel

In receive mode: NCS or LES. In transmit mode: Message or Signalling. Changing channel: Retuning.

Normally: Free, Login, Transmission, Position report, or Receiving message; but there are others.

Hardware id.

The calling code for the system.

We keep this set to none.

Up to 3 tasks the transceiver is waiting to perform, note that transmissions have the lowest priority.

Option 8 - Show system messages

The Satcom-C transceiver produces status messages in response to many events, most of which are acted on by H_SATCOM, and all of which are copied to H_SATCOM.LOG. This display shows the last 23 lines of messages from the transceiver. Except for messages which were found stored in the transceiver when H_SATCOM is started, all messages are preceded by the time they were sent. Most of the messages are self-explanatory, and there is no need for any action to be taken in response to them - H_SATCOM takes action where necessary.

Possible problems

No response from H_SATCOM - The H_SATCOM process is started when HORACE boots and should always be running. Very occasionally H_SATCOM may be busy processing data, but if it fails to respond to requests from the display program for more than a minute, then it is probable that H_SATCOM has got hung; restart it with H_STOP H_SATCOM and H_START H_SATCOM.

No response from Satcom-C transceiver - This indicates that H_SATCOM is unable to get a response from the Satcom-C transceiver. Check that the Satcom-C CB on aft core console EDB is on, that the power light on the front of the transceiver is lit, that HORNET is on and the port 7 light is flashing at least once every 8s. If all these are OK then the RS232 connection may be broken, or possibly HORACE or HORNET are not set up properly.

Message transmission fails - Message transmission may fail for a number of reasons, the most obvious being if line of sight between the aircraft antenna and the satellite is lost, by manoeuvre or by tail masking. Transmission will also fail if the system loses its GPS position during transmission. If the message is to be forwarded from MRF then matching and up-to-date address files must be used on HORACE and the base station. Note that it takes about two minutes to set up, send, and close down a transmission, however short. The system does not try to retransmit failed messages. The Flight Manager should use the 'Resubmit' option if they still want to send the message.

Test message to XV208 fails to return - It may take upto 10 minutes before a test message sent from the aircraft is returned. If the message transmission is shown as being successful, then it has reached the groundstation (normally Goonhilly), so the problem may lie at the MRF base station. In this case try using the "Emergency routing to mobile" option which causes the ground station to return the message to the aircraft without sending it to MRF. If even this option fails then, assuming the message transmission was successful, you will need to seek specialist advice.

No ocean region shown in Option 7 - This may occur if the system can't see any Inmarsat satellites, or if it is changing between satellites. This should be a temporary condition until a satellite becomes visible, however no messages can be transmitted or received until a link has been established to one of the ocean regions.

GPS not 2-D or 3-D in Option 7 - If the GPS is not indicating 2-D or 3-D then it is unable to fix its position, either because less than 3 satellites are useable or because the PDOP is greater than 286. Under these circumstances the system will not be able to carry out any transmissions. Satcom menu option 99 gives access to a menu with the option to initialise the Satcom GPS system with the current Navstar XR5 values, and this should be tried if the GPS is failing to startup or lock on.

H_SATCOM.LOG - The H_SATCOM.LOG file on HORACE provides a record of the setup of the Satcom-C tranceiver, details of all messages sent and received by the system, and a record of all status and error events. There is no need to consult it routinely, but it may be useful to track down any faults with the system.

Message length - The maximum length of message that a Flight Manager can send is 20 lines. The maximum length message that the system can receive is 50 lines or 4000 characters, whichever is smaller.

Receipt confirmation - The system provides no confirmation that a message has been actually reached its destination or receipient. A "Successful" entry in the transmission log simply means that the message has been received by a ground station. If you want to be sure that a message has reached its destination you should ask the receipient to send a reply back to the aircraft.

EGC messages - H_SATCOM tries to disable the reception of Enhanced Group Call messages, which include such things as Inmarsat status messages and gale warnings. They may occasionally appear in the Satcom message log, particularly if the Satcom-C transceiver has been on without HORACE running. There is no general mechanism for looking at the contents of these messages, although they may be listed in the H_SATCOM.LOG file on HORACE.
