Description

The HORSEFLY terminal server is mounted near the top of the rear of the aft core console, together with its mains-to-DC converter. It provides eight RS423 input/output ports to HORACE, to which it is connected by 10Base2 thin wire Ethernet cable. Eight 9-pin D connectors, labelled P1 to P8, are provided on the DECserver panel (H815) for connecting to the HORSEFLY ports.

Only non-critical systems are attached to HORSEFLY, so that even if HORSEFLY is not working HORACE will retain its most important recording and display capability.

HORSEFLY has to down load its operating software from HORACE each time it is switched on; however once this is loaded it remains in place even when HORACE is switched off or rebooted.

There are a set of eleven green lights on the front of HORSEFLY which give a useful indication of its current state, and also the state of the systems attached to HORSEFLY's ports.

Switching on
  1. Ensure the UPS is supplying 230V 50Hz power.
  2. Forward core console, UPS distribution (H831A), S7-S8 - ON.
  3. Observe the power up sequence on the 11 indicator lamps (from left to right - 87654321#C@):
    1. The @ light comes on immediately to indicate power is present.
    2. The port lights (8-1) come on one at a time until all self tests are complete and the C light comes on. This takes about 25 s.
    3. The # light comes on if the Ethernet circuit is OK.
    4. The # light flashes and the port lights 8 to 5 come on in turn while the server tries to download its software from HORACE.
    5. The # light stays on permanently once the server has its software. This is typically 2.5 mins after HORACE is switched on, or 40 s after HORSEFLY is switched on if HORACE is already on.
    6. Individual port lights now come on when the corresponding HORACE processes make connections to them.

Pre-flight and In-flight checks
  1. There is no need for pre-flight or in-flight checks unless there are doubts about system operation; in which case check the HORSEFLY indicator lamps as shown for Normal Operation.

Post-flight
  1. No special action, HORSEFLY is always powered when the UPS and UPS distribution S7-S8 are on.

Note that processes may not make connections immediately the server becomes available; some processes only check the availability of the server on an occasional basis. Also the ports that are being used for keypads only may not start until the appropriate graphics box is also available.

Most HORACE processes operate with no handshaking. Therefore they send data out through the HORSEFLY ports without checking that any system is connected to them or working.

Possible problems
  1. The @ (power) light does not light - no power getting through. Check the UPS and S7-S8 CB are on and cables to/from the server power converter and server are in place.
  2. The C (selftest) light does not light - probably a hardware problem with the server - it will need to be replaced, but note the status of the eight port lights since these will indicate the nature of the fault. If all port lights except 1 and 2 are lit, then there is a problem with the Ethernet connection to the server.
  3. The # (network) light does not light - probably a problem with the Ethernet circuit - either open circuit or not terminated properly.
  4. The # (network) light stays flashing - failing to download the software. HORACE must be up and running (ie DECnet must be running). Even when HORACE is running there may be a delay of up to a minute as the server tries different protocols and waits before retrying. If the server has recently been changed the problem may be that the new server and its Ethernet address has not been entered in the DECnet database.
  5. If a particular port light does not come on or no communication takes place through it (light doesn't flash) then try stopping and starting the HORACE process associated with that port. If this has no effect switch off the server, switch it on again, and then again try stopping and starting any HORACE processes which don't start working by themselves.
